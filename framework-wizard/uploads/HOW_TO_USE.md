# RN Framework Key Docs — How to Use This Folder

## Overview

When someone has an idea for a Regenerative Neighborhood project, use the documents in this folder to guide them through the full framework — from vision to execution.

---

## Step-by-Step Process

### Step 1: Read the Execution Plan First

Start with **`execution-plan-rnf-v2.md`** — this is your master guide. It contains the complete implementation roadmap for the Regenerative Neighborhood Framework.

Read it fully before doing anything else. It will give you:
- The full project lifecycle
- Key phases and milestones
- Which documents to reference at each stage
- How to sequence the work

### Step 2: Reference Other Documents as Questions Arise

As the vision holder works through the execution plan, other documents answer specific questions:

| Document | When to Use |
|----------|-------------|
| **`community-alchemy-playbook.md`** | When helping someone articulate their community vision, values, and purpose |
| **`agent-workbook.md`** | When working through the 67 onboarding questions (who, what, where, how) |
| **`EXECUTIVE-SUMMARY.md`** | When someone needs a high-level overview before diving deep |
| **`RCOS.md`** | When designing the operating system for the community |
| **`CLIPS.md`** | When building expert rules or decision logic for the community |
| **`execution-plan-regen-neighborhood.md`** | For a lighter-weight planning view (shorter version) |
| **`Spiral-Framework_*.docx`** | When sequencing the project phases or stakeholder development stages |

### Step 3: Walk Them Through the Execution Plan

Use the execution-plan-rnf-v2.md as your script. Guide the vision holder through each phase:

1. **Foundation** — vision, purpose, values
2. **Design** — land, systems, governance
3. **Build** — infrastructure, community formation
4. **Launch** — first members, operations
5. **Scale** — growth, network effects

At each phase, reference the relevant document from the table above.
