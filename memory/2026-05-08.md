# Memory — 2026-05-08

## SPEC Duplicate "As a" Fix (Vitali, Topic 2088)
Root cause: `build.py` hardcoded `As a regenerative community member` prefix for ALL feature descriptions, causing double "As a" when the description already started with that phrase.

Fix: Added condition in `build.py` to only prepend "As a regenerative community member" when the description doesn't already start with it. Committed as `8d13ab0` to `genesis-zero-bot-spec`.

Discovery: The raw `.feature` files were clean — the bug was in the HTML generator, not the source files. The `build.py` creates `docs/{id}-{slug}.feature` and prepends the prefix unconditionally.

## Arch Repo Pages + Archi Report Fix
Three changes made to `genesis-zero-bot-arch`:

1. **model/ symlink → model-example/** — deploy-archi-report action needs `model/folder.xml` at project root (Archi directory format). Our `.archimate` single-file models in `models/` don't have this. Linked `model/` → `model-example/`.

2. **Pages switched from `workflow` → `legacy`** — deploy-archi-report pushes to `gh-pages` branch. Pages was set to `workflow` (artifact) mode, incompatible. Changed to `legacy` (branch-based) via `pages -X PATCH`.

3. **Workflow rewrote** (025e522): Docker Archi generates HTML → deploy-archi-report pushes to `.archi_report/` on gh-pages → custom `index.html` at gh-pages root alongside Archi output.

Key insight: Archi `--import.archi` followed by `--saveModel` fails with "Model not set" because import doesn't set the model for subsequent save. The proper workflow is `--modelrepository.loadModel <path>` which loads directly into Archi's in-memory model, then HTML generation works.

## Critical: Remote Rename Warning
Both `arch` and `archimate` remotes point to `genesis-zero-bot-archimate.git` — the old name. After rename to `genesis-zero-bot-arch`, the remote URL should be `regentribes/genesis-zero-bot-arch.git`. The push succeeded with a "repository moved" warning. Should update remote URL in local clones.

## Vitali's Full Repo Audit
He reported "multiple inconsistencies found in every of four projects." The SPEC build.py was the only actual bug. The other repos (arch, adrs, wiki) don't have `.feature` files. The arch repo had Pages misconfiguration (workflow vs legacy), now fixed.
