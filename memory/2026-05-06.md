## ISPEC-001 — Integral Bootstrap Specification (Vitali, 2026-05-06)

**Source:** Topic 1958, Regen Tribe Collective Network  
**Status:** REV 1.0 COMPLETE — 5 sections, 40 addenda, ready for implementation review

### DECISION
Every new member boots through a defined bootstrap sequence:
1. Install client binary
2. Verify factory signature
3. Establish UNTP credentials through CDS deliberation
4. Connect to local fleet
5. Activate ITC ledger participation

### RATIONALE
- Defined bootstrap prevents inconsistent member states (partial bootstrap → degraded states hard to diagnose)
- Guarantees identical baseline configuration for every completing member
- Ensures CDS knows about every member before governance participation

### REJECTED ALTERNATIVES
- Ad hoc: inconsistent states requiring manual remediation
- Managed services: single points of failure
- Self-service without CDS deliberation: bad actors can join without community oversight

### SPECIFICATION CONSEQUENCE
- Single shell script / executable running end-to-end
- Every step logged to ITC ledger
- Failed steps block progression and publish ITC events
- FRS monitors bootstrap completion rates
- CDS deliberates on failures above defined threshold rate

### Subsystems referenced
- ITC (Integrity Trust Computation) ledger
- CDS (Collective Decision System) deliberation
- FRS (Fleet Registry System)
- UNTP credentials
- Factory signature verification
